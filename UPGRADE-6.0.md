# Upgrading to `bootstrap_form` 6.0

`bootstrap_form` 6.0 generates mark-up for Bootstrap 6. Bootstrap 6 renames and restructures many of the form-related classes, so the mark-up generated by this gem changes accordingly. If your application styles or scripts against the generated mark-up, or if you pass Bootstrap utility classes to the helpers, you will need to make changes.

## Bootstrap 6 Changes

Upgrading `bootstrap_form` to version 6 means you must upgrade your whole application to Bootstrap 6. Read the [Bootstrap 6 migration guide](https://github.com/twbs/bootstrap/blob/v6-dev/skills/bootstrap-v5-v6-migration/SKILL.md) and the [Bootstrap 6 forms documentation](https://v6-dev--twbs-bootstrap.netlify.app/docs/6.0/forms/overview/) to see what changes you have to make to your views.

## Changes to Generated Mark-up

### Form Group Wrappers

Bootstrap 6 introduces the [`form-field` wrapper](https://v6-dev--twbs-bootstrap.netlify.app/docs/6.0/forms/field/), a CSS-grid container that lays out a control together with its label, help text, and validation feedback. `bootstrap_form` now uses it as the default form group wrapper:

- Vertical form groups are wrapped in `class="form-field mb-3"` (was `class="mb-3"`).
- Collections rendered with `group_around_collections = true` are wrapped in `class="form-group mb-3"`, Bootstrap 6's wrapper for multiple `form-field`s under a shared label.
- Horizontal groups (`mb-3 row`), inline groups, and floating label groups (`mb-3 form-floating`) are unchanged, because they use Bootstrap's grid or the `form-floating` component for layout instead.
- A custom `wrapper_class`/`wrapper: { class: ... }` still *replaces* the default classes. Include `form-field` in your custom class if you want to keep Bootstrap 6's field layout.

### Help Text

Help text is now rendered as `<small class="form-text">` without the `text-muted` class. Bootstrap 6 removes `text-muted` (its replacement is `fg-secondary`), and `form-text` styles itself.

### Selects

Bootstrap 6 styles `<select>` elements with `.form-control` instead of `.form-select`. All select helpers (`select`, `collection_select`, `grouped_collection_select`, `time_zone_select`, and the date/time select helpers) now generate `class="form-control"`.

### Checkboxes and Radio Buttons

Bootstrap 6 replaces the `.form-check` component with the `.form-field` wrapper and dedicated control classes:

- The wrapper `<div>` is now `class="form-field"` instead of `class="form-check"`.
- Checkbox inputs get `class="check"` instead of `class="form-check-input"`.
- Radio inputs get `class="radio"` instead of `class="form-check-input"`.
- Labels no longer get the `form-check-label` class (a plain `<label>` is used).
- Inline checks (`inline: true`) use the `d-inline-grid me-3` utilities on the wrapper instead of `form-check-inline`, since Bootstrap 6 has no dedicated inline class.

### Switches

Bootstrap 6 gives switches their own structure. `check_box ..., switch: true` now generates:

```html
<div class="form-field mb-3">
  <div class="switch">
    <input name="user[remember_me]" type="hidden" value="0">
    <input id="user_remember_me" name="user[remember_me]" role="switch" switch="switch" type="checkbox" value="1">
  </div>
  <label for="user_remember_me">Remember me</label>
</div>
```

instead of the v5 `form-check form-switch` wrapper with a `form-check-input` class.

### Range Fields

Bootstrap 6 wraps range inputs in a component element. `range_field` now generates:

```html
<div class="form-range">
  <input class="form-range-input" ... type="range">
</div>
```

instead of a bare `<input class="form-range">`.

### Floating Labels

Bootstrap 6 requires the `<label>` to come *before* the control inside `.form-floating` (the CSS relies on `:has()`). `bootstrap_form` now renders floating labels before the field instead of after it.

### Buttons

Bootstrap 6 removes the `.btn` base class and single-class color variants in favour of composed style + theme classes:

- `submit` and `button` default to `class="btn-solid theme-secondary"` (was `btn btn-secondary`).
- `primary` defaults to `class="btn-solid theme-primary"` (was `btn btn-primary`).

If you pass your own `class:` to these helpers, update it to the new Bootstrap 6 button classes yourself.

### Horizontal Forms and Grid Classes

Bootstrap 6 changes responsive class syntax from suffix to prefix: `.col-sm-2` becomes `.sm:col-2`, `.offset-sm-2` becomes `.sm:offset-2`.

- The default `label_col` is now `"sm:col-2"` (was `"col-sm-2"`).
- The default `control_col` is now `"sm:col-10"` (was `"col-sm-10"`).
- If you pass custom `label_col`/`control_col` values, use the new syntax; the automatic offset calculation for label-less groups understands only the new syntax (e.g. `sm:col-2` → `sm:offset-2`).
- Inline layout labels get `class="form-label sm:me-2"` (was `form-label me-sm-2`).

### Alerts

`alert_message` now defaults to `class="alert theme-danger"` instead of `class="alert alert-danger"`, following Bootstrap 6's composed theme classes. Pass `class: "alert alert-danger"` explicitly if you need the old mark-up.

## New Features

### Datepicker Helper

Bootstrap 6 introduces a datepicker component. The new `datepicker_field` helper generates a text field with `data-bs-toggle="datepicker"` and renders the `:datepicker` option hash as `data-bs-*` configuration attributes. See the README for details.

## Unchanged

The following classes are unchanged in Bootstrap 6 and continue to be generated as before: `form-label`, `form-text` (now without `text-muted`), `form-control` (and its `-sm`/`-lg`/`-color`/`-plaintext` variants), `form-floating`, `input-group`, `input-group-text`, `col-form-label`, `is-invalid`/`is-valid`, `invalid-feedback`/`valid-feedback`, and `visually-hidden`.
